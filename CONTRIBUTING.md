## 🚀 Getting Started

### 1. Fork المشروع

```bash
git clone https://github.com/moelshahat2002/spaceappsdamietta.git
cd spaceappsdamietta
```

### 2. أنشئ branch جديد

```bash
git checkout -b feature/your-feature-name
# أو
git checkout -b fix/your-bug-fix
```

### 3. عدّل الملفات وأضف تحسيناتك

```bash
# شغّل السيرفر لاختبار التغييرات
npm start

# افتح http://localhost:8080/Love_Site_Final.html
```

### 4. Commit التغييرات

```bash
git add .
git commit -m "✨ Add amazing feature
- Detail 1
- Detail 2"
```

### 5. Push و Open PR

```bash
git push origin feature/your-feature-name
```

## 📋 Commit Message Guidelines

استخدم Conventional Commits:

```
✨ feat: Add new feature
🐛 fix: Fix a bug
🎨 style: CSS/styling changes
📝 docs: Documentation updates
♻️ refactor: Code refactoring
⚡ perf: Performance improvements
✅ test: Add tests
🔧 chore: Build process or dependencies
```

## 🎯 Best Practices

### JavaScript
```javascript
// ✅ Good - واضح ومنظم
function createFloatingHeart() {
    // Implementation
}

// ❌ Bad - غير واضح
function ch() {
    // Implementation
}
```

### CSS
```css
/* ✅ Good - منظم مع comments */
/* ==================== BUTTONS ==================== */
.start-btn {
    /* styles */
}

/* ❌ Bad - غير منظم */
.btn { }
```

### HTML
```html
<!-- ✅ Good - Semantic & Accessible -->
<button 
    class="start-btn" 
    onclick="openLove()" 
    aria-label="ادخلي إلى الموقع">
    اضغطي هنا ❤️
</button>

<!-- ❌ Bad - غير accessible -->
<div onclick="openLove()">Click</div>
```

## 🐛 الإبلاغ عن الأخطاء

عند إيجاد خطأ:

1. **تحقق** إذا كان الخطأ موجود بالفعل
2. **صف** الخطأ بوضوح:
   - ما الذي توقعته؟
   - ما الذي حدث بدلاً منه؟
   - خطوات إعادة الإنتاج
   - متصفح وإصدار
3. **أرفق** صور أو فيديو إذا أمكن
4. **اقترح** حلول إذا كان لديك
